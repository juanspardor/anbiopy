Anbiopy Library Documentation¶

anbiopy is a python library designed for microbiota data analysis when incorporating other types of variables e.g., anthropometric data.

For installation use:

pip install anbiopy

This file contains the documentation for all functions available in version 0.1.3

Recommended libraries to install and use¶

anbiopy uses libraries like pandas, numpy and scikit-bio. However, for data manipulation and visualizations it is useful to import the following libraries

In [1]:
import anbiopy as ab
import pandas as pd
import seaborn as sns

4. alpha_diversity()¶

Calculates the alpha diversity of each individual for a given set of metrics.

Parameters:

  • otus: OTU Observation DataFrame. It can include an ID column
  • metrics: A String list containing the desired metrics. Supported metrics are: "shannon", "sv-richness", "heip_e", "simpson".
  • includes_id (Optional): Boolean that indicates if the pData includes an ID column. Default value = False
  • rarefy (Optional): Boolean that indicates if the user desires to perform rarefaction on the dataset before calculating alpha-diversities. Default value = True
  • seed (Optional): Random seed for replicability. Default value = 23

Output: A DataFrame containing each desired metric for each sample. If the user included an ID column it will also be included and named "Sample_ID"

5. beta_diversities()¶

Calculates the alpha diversity of each individual for a given set of metrics.

Parameters:

  • otus: OTU Observation DataFrame. I can include an ID column
  • metrics: A String list containing the desired metrics. Supported metrics are: "weighted_unifrac", "unweighted_unifrac".
  • tree_string: String that represents the phylogenetic tree of the OTUs
  • includes_id (Optional): Boolean that indicates if the pData includes an ID column. Default value = False
  • rarefy (Optional): Boolean that indicates if the user desires to perform rarefaction on the dataset before calculating alpha-diversities. Default value = True
  • seed (Optional): Random seed for replicability. Default value = 23

Output: A list that in each position has the beta diversity values for its corresponding metric. Each position contains a list of the elements in the upper triangular of the NxN beta diveristy (these elements do not include the diagonal). If the complete matrix is needed, use convert_vector_beta_matrix().

8. create_graphic()¶

Creates different types of graphics depending on parameter input. The type of graphics are explained in the graphic_type parameter.

Parameters:

  • Y: Observations of objective variable
  • X: Observations of explicative variables
  • graphic_type: Number indicating the type of graphic to be created.
    • 1: Y vs. Numeric X -> Line plot
    • 2: Y vs X pData -> Violin plot
    • 3: Y vs Numeric X (first column) and Categoric X (second column) -> Multiple line plot
  • save_file (Optional): Boolean to indicate if the created graph should be saved. Default value = False
  • file_name (Optional): String for the file name in case the graphic should be saved. Default value = " "

Output: No return value. It prints the image on the cell and saves it if necessary

9. anova_test()¶

Creates, runs and evaluates an ANOVA model using selected variables. It validates the 3 assumptions of the ANOVA model: Homocedasticity, Error Normality, Independence. The function call works similar to aov() in R, in the sense that it can receive an explicative variable matrix and the user indicates which ones to use.

Parameters::

  • Y: Observations of the objective variable. This should be a numeric variable
  • data: DataFrame containing multiple variables.
  • X_columns: List of strings containing the names of the variables in data to be used in the model. All these variables should be categoric.
  • alpha (Optional): Alpha value for statistical tests. Default value = 0.05

Output: No return value. It prints a message with the following information:

  1. ANOVA model results

  2. Homocedasticity test and result

  3. Error Normality test and result

  4. Independe test and result

  5. Assumption conclusion.

    5.1 If all assumptions are validated -> Confirmation 5.2 If at least one assumption is not validated -> Indication and possible transformation (depends on number of explicative variables)

# ONE-WAY ANOVA
ab.anova_test(Y = alpha_diversity_test['shannon'],

10. multi_level_factors()¶

For a given objective variable, identifies possible important predictors at different taxonomic levels using Random Forest predicting models. It produces a grid of the most important predictors at each taxonomic level.

Basic logic of execution:

  1. For each selected taxonomic level:

    1.1 Groups all OTU observations based on their group within the taxonomic level

    1.2 Creates a RF model using the grouped OTU observations. Be default all RF models are created using sqrt(No. Columns) variables

    1.3 Selected the top max_rows groups and includes them in the level's plot.

  2. Returns the list of importances for each level

Parameters:

  • otus: DataFrame of OTU observations
  • otu_taxonomy: DataFrame of taxonomic classification of each OTU. It must contain the same columns as the output of taxonomy_df() i.e., it must have Kingdom, Phylum,..., columns.
  • Y: Observations of the objective variable
  • Y_name: Name of the objective variable, for the title of the plots.
  • levels: List of strings containing the taxonomic levels to be used. Supported values are "Kingdom", "Phylum", "Class", "Order", "Family", "Genus", "Species"
  • classification (Optional): Boolean to indicate if the predicting task is classification or not. Default value = False
  • includes_ids (Optional): Boolean to indicate if otus contains an ID column. Default value = False
  • max_rows (Optional): Number of top predictors to include in each plot. Default value = 10
  • relatve (Optional): Boolean to indicate if relative abundances should be calculated for otus. Default value = True
  • seed (Optional): Randon seed for replicability of results. Default value = 23
  • save_file (Optional): Boolean to indicate if the final plot should be saved. Defalut value = False
  • file_name (Optional): String for the file name of the image should it be saved

Ouputs

  • Prints the grid of plots. If indicated, saves the grid of plots.
  • Returns a list of DataFrames, each one corresponding to its respective pLevels value. Each DataFrame contains the complete table of Predictor-Importance values

13. friedman_test()¶

Non-parametric statistical test. It tests if the distribution of an objective variable is the same between different groups.

Requirements:

  • Only 1 explicative variable
  • All groups must have the same number of observations
  • At least 3 groups

Parameters:

  • Y: Observations of the objective variable. Must be numeric
  • X: Observations of the explicative variable. Must be categoric
  • alpha (Optional): Alpha value for statistical tests. Defalut value = 0.05

Outputs: No return value. It prints the result of the test

In [28]:
